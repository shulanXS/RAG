# Sample Document for RAG Testing

## 第一章：RAG 系统概述

检索增强生成（Retrieval-Augmented Generation, RAG）是一种结合了信息检索和大语言模型的技术架构。

RAG 系统的核心思想是在生成答案之前，先从外部知识库中检索相关信息，然后将这些信息作为上下文提供给大语言模型。

## 1.1 RAG 的发展历程

2023年：RAG 概念首次被提出，主要用于解决大语言模型的幻觉问题。

2024年：RAG 技术快速发展，混合检索、重新排序、多跳推理等技术开始成熟。

2025-2026年：Agentic RAG 成为主流，RAG 系统开始具备自主规划、工具调用和多步推理能力。

## 1.2 为什么需要 RAG

大语言模型虽然具备强大的生成能力，但存在以下问题：

1. **知识过时**：模型的知识截止到训练日期，无法获取最新信息。
2. **幻觉问题**：模型可能生成看似合理但实际错误的内容。
3. **领域知识不足**：通用模型对特定领域的专业知识掌握有限。
4. **无法溯源**：模型无法提供答案的信息来源。

RAG 通过在生成过程中引入外部检索，有效解决了以上问题。

## 第二章：RAG 核心技术

### 2.1 文档分块（Chunking）

文档分块是将长文档切分为小片段的过程。分块策略直接影响检索质量。

常见的分块方法包括：
- **固定大小分块**：按预设的 token 数量切分
- **递归字符分块**：按层级分隔符（段落、句子、单词）递归切分
- **语义分块**：根据语义相似度动态检测断点
- **层级分块**：按文档标题结构切分

### 2.2 向量检索（Vector Retrieval）

向量检索使用文本嵌入模型将文本转换为向量表示，然后在向量空间中进行相似度搜索。

主流的嵌入模型包括：
- **OpenAI text-embedding-3-large**：通用场景
- **Voyage AI voyage-3-large**：英文+代码场景最优
- **BGE-M3**：开源多语言首选

### 2.3 混合检索（Hybrid Search）

混合检索结合了关键词检索（BM25）和向量检索的优点：

- **BM25**：擅长精确匹配，如 SKU、合同号等
- **向量检索**：擅长语义理解，如 paraphrase
- **RRF 融合**：将两种结果按排名融合，无需归一化

## 第三章：生产级 RAG 架构

### 3.1 Agentic 编排

Agentic RAG 通过中央编排器协调多个专业组件：

1. **查询路由器**：判断查询复杂度
2. **检索引擎**：执行混合检索
3. **重排序器**：Cross-Encoder 精排
4. **记忆库**：追踪 claim-evidence 链路

### 3.2 评估体系

RAGAS 评估框架定义了五大核心指标：

| 指标 | 定义 |
|------|------|
| Faithfulness | 答案是否被上下文支撑 |
| Answer Relevancy | 答案是否直接回答问题 |
| Context Precision | top-K 中相关块的比例 |
| Context Recall | 上下文覆盖必要信息的程度 |
| Answer Correctness | 与参考答案的一致性 |

### 3.3 语义缓存

语义缓存通过存储相似查询的答案，显著降低 LLM 调用成本：

- **命中阈值**：cosine similarity ≥ 0.92
- **典型命中率**：30-60%（取决于查询重复率）
- **成本节省**：40-80%

## 结论

企业级 RAG 系统需要综合考虑检索质量、生成质量、系统延迟、运维成本等多个维度。

2026 年的最佳实践是：混合检索 + Cross-Encoder 重排序 + Agentic 编排 + RAGAS 评估。
